# Описание BPMN/UML логики расчета стоимости пошива

## 1. Роли и сущности

- `Пользователь`: владелец аккаунта и правил расчета.
- `Чат`: контекст общения/сценарий расчета внутри конкретного пользователя.
- `Настройки пользователя`: базовые цены, надбавки, скидки по партиям.
- `Расчет`: результат вычисления стоимости заказа, сохраненный в чате.

## 2. BPMN-сценарий "Первичный вход и настройка"

### Участники (Swimlanes)
- Пользователь
- Система

### Последовательность
1. `Start event`: пользователь открывает сервис.
2. Пользователь проходит регистрацию/авторизацию.
3. Система создает профиль пользователя.
4. Пользователь заполняет правила ценообразования:
   - базовые цены на изделия;
   - надбавки (усложнения);
   - скидки по диапазонам партии.
5. Пользователь нажимает `Сохранить`.
6. Система валидирует данные.
7. `Gateway`:
   - если валидация не пройдена: вернуть ошибки в форму;
   - если пройдена: сохранить настройки.
8. `End event`: настройки пользователя сохранены.

## 3. BPMN-сценарий "Повторный вход и расчет"

### Участники (Swimlanes)
- Пользователь
- Система

### Последовательность
1. `Start event`: пользователь открывает чат расчета.
2. Пользователь вводит параметры заказа:
   - тип изделия;
   - список усложнений;
   - размер партии.
3. Система загружает настройки пользователя.
4. Система выполняет расчет:
   - определяет базовую цену;
   - применяет надбавки;
   - умножает на количество;
   - применяет скидку диапазона.
5. Система сохраняет расчет в истории чата.
6. Система возвращает итог и детализацию.
7. `End event`: пользователь получает стоимость заказа.

## 4. UML Use Case (кратко)

### Актор
- `User`

### Use Cases
- `Manage Pricing Settings`
- `Calculate Order in Chat`
- `View Chat Calculation History`

### Связи
- `Calculate Order in Chat` <<include>> `Load User Settings`
- `Calculate Order in Chat` <<include>> `Persist Calculation`

## 5. UML Sequence (сервисный слой)

Участники:
- `User`
- `HTTP API`
- `CostingService`
- `SettingsRepository`
- `ChatRepository`

### Последовательность сообщений
1. `User -> HTTP API`: POST `/calculate` с `OrderInput`.
2. `HTTP API -> CostingService`: `CalculateInChat(userID, chatID, order)`.
3. `CostingService -> SettingsRepository`: `GetSettings(userID)`.
4. `SettingsRepository --> CostingService`: `UserSettings`.
5. `CostingService`: вычисляет `CalculationResult`.
6. `CostingService -> ChatRepository`: `AppendCalculation(result)`.
7. `ChatRepository --> CostingService`: OK.
8. `CostingService --> HTTP API`: `CalculationResult`.
9. `HTTP API --> User`: JSON с итогом расчета.

## 6. Формула расчета

Обозначения:
- `B` — базовая цена за единицу изделия.
- `s_i` — процент i-го усложнения.
- `n_i` — количество повторов i-го усложнения.
- `Q` — размер партии.
- `D(Q)` — процент скидки для диапазона партии.

Формулы:
1. Надбавка за единицу:
   `S = Σ(B * s_i/100 * n_i)`
2. Цена за единицу с надбавками:
   `P = B + S`
3. Стоимость до скидки:
   `SUBTOTAL = P * Q`
4. Скидка:
   `DISCOUNT = SUBTOTAL * D(Q)/100`
5. Итог:
   `TOTAL = SUBTOTAL - DISCOUNT`

Пример:
- `B = 5000` (пиджак)
- усложнения: 3 раза по 20%
- `Q = 15`, `D(15) = 5%`

Тогда:
- `S = 5000 * 0.2 * 3 = 3000`
- `P = 8000`
- `SUBTOTAL = 120000`
- `DISCOUNT = 6000`
- `TOTAL = 114000`
